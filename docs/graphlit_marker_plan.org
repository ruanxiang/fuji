* Objective
Create a seamless Emacs workflow that allows for natural language interaction with academic papers, supporting both textual reasoning (via Graphlit RAG) and visual explanation of figures (via Multimodal LLM & Marker extracted images).

* Architecture Overview
The system is designed with a *Pluggable Provider Architecture* to allow future-proofing (e.g., replacing Marker with MinerU, or Graphlit with a local RAG).

- **Search & Selection**: ~org-ref~ (BibTeX) as the entry point.
- **Parser Layer (Pluggable)**: 
  - Current: ~Marker~ (local).
  - Interface: Accepts PDF, outputs Markdown + Assets.
- **RAG Backend (Pluggable)**: 
  - Current: ~Graphlit~ (cloud).
  - Interface: Accepts Markdown, provides semantic search/chat.
- **UI & Interaction**: ~gptel~ as the unified interface, acting as an orchestrator.
